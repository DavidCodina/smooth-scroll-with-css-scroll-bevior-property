By setting overflow: hidden on body, and overflow: scroll on main (i.e., the scroll container), we reassign the scrollbar to the appropriate parent container, and this allows us to implement the scroll snap feature.

However, if you set it up this way, you have to be sure to implement the scroll snap feature. In other words, this is not a good way to set up smooth scroll. Why? Because on mobile devices the scroll won't glide.

Personally, I find this feature to be kind of annoying...

That said, overall the scroll-behavior property is a great feature. The only issue is browser compatibility. See caniuse.com. That said, if you try to use it in Safari, it won't break the page. It just doesn't smooth scroll.
